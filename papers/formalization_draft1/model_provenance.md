## Principles for paper wording

- The mathematical claim should always be phrased as a claim about the checked
  Lean artifact, not as a claim about any model's reliability.
- Model names should appear in an assistance/provenance statement, not in the
  abstract or as the main scientific contribution.
- Repository comments and git history support several model attributions, but
  not all attributions equally.  In particular, GPT-family commits were often
  not named in commit metadata, so GPT provenance must rely on inline comments
  and external session logs.
